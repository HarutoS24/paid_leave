# paid leave

## 概要

有給管理アプリ。  
作者である佐藤陽人の go と react の技術力を示すデモである。  
フロントエンド・バックエンド・インフラ環境の構築は全て佐藤陽人が一人で行なっている。

## 要件

- docker compose を使い、簡単に再現可能な db・フロントエンド・バックエンドサーバー環境の構築を行うこと。
- CI/CD 環境を構築し、

  - コードの規約チェック
  - コードの自動テスト
  - 自動デプロイ

  を github actions を使って実現すること。

## 使用技術

- go 1.24.2
- vite 7.1.7
- react 19.1.1
- docker 28.5.1

## ディレクトリ構成

```text
.
├── backend　バックエンドのソースコード。
├── docker
├── cd デプロイ関係のディレクトリ。
│   ├── backend
│   ├── db
│   └── frontend
├── ci 自動チェック関係のディレクトリ。
│   ├── backend
│   └── frontend
└── frontend　フロントエンドのソースコード。
```

## ローカルでの使用方法

### 1. docker のインストール

- docker をローカル環境にインストールする。
- 推奨バージョン: `v28.5.1`

### 2. env ファイルの設定

- cd ディレクトリ下に環境変数(ポート、db 環境)をセットするための.env.template ファイルがあるので、適切に変更して.env ファイルに名称変更する。

### 3. システムを動作させる方法

1. cd ディレクトリに移動する。
1. make up コマンドを実行すると、docker コンテナのビルドと依存関係の構築が自動で行われる。
1. ブラウザから `http://localhost:<env ファイルで記入したポート番号>` にアクセスすると、フロントエンドとバックエンドの動作が確認できる。
